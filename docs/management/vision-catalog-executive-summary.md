# Умный каталог Аквафор-Pro: поиск по фото и тексту

> Executive summary для руководителя. Технические детали — `docs/features/vision-catalog-search.md`.
> **Статус:** ✅ Phase 1 + Phase 2 закрыты (2 мая 2026) — universal endpoint работает на реальных 155 товарах Аквафор-Pro, обогащённых AI-описаниями фото, готов к интеграции с фронтом.

---

## Что строим

AI-сервис поиска оборудования и услуг водоочистки по **фотографии** или **текстовому запросу** на естественном языке. Встраивается в `prostor-app` (наш веб + Telegram Mini App) и `crm-aqua-kinetics-back` (CRM для менеджеров и инженеров).

**Сценарий №1.** Менеджер получает от клиента фото фильтра в WhatsApp — загружает в CRM, через 3 секунды видит подходящий товар из каталога Аквафор-Pro с ценой, нужными расходниками и услугой монтажа.

**Сценарий №2.** Клиент пишет в чате *«фильтр для жёсткой воды у дачи»* — система находит подходящие умягчители и обратные осмосы, не требуя точного знания названий моделей.

**Сценарий №3** (Phase 2). Клиент описывает проблему *«у меня жёлтая вода»* — AI диагностирует *«железо»*, подбирает обезжелезиватель и предлагает заказать анализ воды.

## Бизнес-ценность

| Что улучшается | Как |
|---|---|
| **Скорость подбора товара** | С 3-5 минут листания каталога до 2-3 секунд AI-поиска |
| **Конверсия лидов** | Быстрый ответ клиенту увеличивает вероятность сделки на 15-30% (исследования B2B) |
| **Cross-sell** | Каждый подобранный фильтр сразу предлагает связанные услуги (монтаж, ТО) и расходники (картриджи) — рост среднего чека |
| **Самообслуживание клиента** | Клиент может сам подобрать оборудование через сайт без участия менеджера — снижает нагрузку на отдел продаж |
| **Обучение новых сотрудников** | Семантический поиск работает даже без знания точных артикулов — стажёры продают с первого дня |
| **Накопление knowledge base** | Каждое распознавание — это AI-описание товара. Со временем накапливается богатый текстовый dataset для будущих фич (диагностика, рекомендации) |

## Технологический стек

| Компонент | Назначение | Лицензия |
|---|---|---|
| **Claude API (Vision)** | Распознавание фото оборудования | Pay-as-you-go ~$0.003/фото |
| **OpenAI Embeddings** | Семантический поиск (превращает текст в математический вектор для сравнения) | Pay-as-you-go ~$0.02/1М токенов |
| **Flowise** | Низкокодовый orchestrator для AI-pipeline (визуальное управление промптами и моделями) | Open-source, self-hosted |
| **PostgreSQL + pgvector** | Хранение каталога и векторных представлений | Open-source |
| **NestJS + Prisma + Docker** | Backend и инфраструктура (наш существующий стек) | Open-source |

Разработка ведётся в свободное время, расходов на нее нет — соответственно в финансовом плане учитываются только реальные **операционные** затраты на AI-сервисы.

## Финансовая модель

### Что компания платит

**Только operational costs внешних AI-провайдеров.** Инфраструктура (сервер, БД, Docker) уже включена в существующий стек CRM — incremental затрат нет. Разработка ведётся в режиме эксперимента / pet-project, без оплаты часов разработчика.

### Реальные замеры — Phase 1 + Phase 2 закрыты (24 апр – 2 мая 2026, 9 дней активной работы с AI-провайдерами)

Снимки billing dashboards двух AI-провайдеров. Первые две недели апреля (Phase 0) были без LLM-вызовов — только chatflow дизайн в Flowise UI и валидация промптов на 6 тестовых фото:

| Метрика | Значение |
|---|---|
| Каталог (текущий snapshot Аквафор-Pro) | **155 товаров** |
| **OpenAI Embeddings** (24 апр – 1 мая, 8 дней) | **$0.02 ≈ 1,6 ₽** |
| **Anthropic Vision** (image-search e2e + multi-image отладка) | **$0.16 ≈ 12,8 ₽** |
| **Совокупно за всю Phase 1 разработку** | **~$0.18 ≈ 14,4 ₽** |

В эту сумму вошли:
- 2 полных переиндексации каталога 155 товаров (PR6 + PR6.5 refactor)
- 1 полная + 1 zero-cost переиндексация при PR9.5 verify (1 мая): первая `$0.00385`, вторая `$0` через RecordManager skip
- Все dev-тесты поиска: text / image / multi-image / combined
- 5 e2e сценариев на Playwright (text / image-only / irrelevant cat / combined / empty)
- Multi-image prompt v2 отладка (3 цикла Vision retest)

**Phase 2 — Vision augmentation на ingest** (2 мая, отдельный шаг):
- Полный re-augment 155 товаров через Claude Haiku 4.5: **$0.395 ≈ 32 ₽**
  один раз (фактический Anthropic billing, в 4× дешевле conservative-оценки)
- Re-embed обновлённых текстов в pgvector: $0.0066 ≈ 0,53 ₽
- E2E demo прогон 7 сценариев (Sonnet runs): $0.016 ≈ 1,3 ₽
- **Итого Phase 1 + Phase 2 за всю разработку: ~$0.49 ≈ 39 ₽**

**Контекст для понимания масштаба:** одна чашка кофе стоит в 5-10 раз дороже, чем 9 дней работы с production AI-моделями на реальных данных каталога 155 товаров.

### Ежемесячный итог при пилоте (после миграции с Telegram mini-app)

**Контекст:** prostor-app сейчас в разработке, мигрирует с Telegram mini-app (закрыт в РФ для коммерческого доступа). Каталог поиска **открытый** — без логина, доступен любому клиенту. Менеджеры пользуются тем же endpoint'ом из CRM. Точное соотношение фото-vs-текст запросов и реальный traffic-поток будут известны после первой недели после запуска — сейчас даём вилку прогнозов.

| Категория | ₽/мес |
|---|---|
| Переиндексация каталога (cron 4ч, до PR9.5) | **~57** |
| После PR9.5 RecordManager skip-if-unchanged ✅ | **~0,6** |
| Поиск (см. сценарии ниже) | вариативно |
| Инфраструктура (уже оплачена в составе CRM) | 0 |

### Прогноз роста стоимости — три сценария после запуска

Cost зависит от объёма traffic'а **и** доли фото-запросов. Image-search в **12 000 раз** дороже текстового — поэтому матрица.

| Сценарий | Поисков/день (text/image) | Cost/мес |
|---|---|---|
| **Стартовая база** (миграция первых клиентов с TG, малый traffic) | 200 / 100 | **~720 ₽** |
| **Активный пилот** (вся клиентская база prostor-app + менеджеры CRM) | 1 000 / 500 | **~3 600 ₽** |
| **Зрелая платформа** (после полной миграции с TG + органика) | 5 000 / 2 500 | **~18 000 ₽** |

Ассумпция по % image: **33%** запросов фотографиями (фото своего фильтра/картриджа — естественный жест клиента). По мере накопления реальных данных скорректируем.

**Заложенные оптимизации после первой недели после запуска:**
- **SHA256-кэш повторных image-запросов** — клиент 2-3 раза фотографирует тот же фильтр → второй и далее ответы из кэша, $0. Ожидаемая экономия 30-50% на image (до запуска не реализован).
- **Redis-кэш text-embedding запросов** — частые формулировки повторяются, embedding'и дешёвые но при 5К запросов/день экономит копейки → не приоритет.
- **Per-IP rate limit** (см. раздел рисков) — отрезает abuse, не сокращает legitimate cost.

### Ценность — главный канал клиенты, вторично менеджеры

**Главный канал: сохранение коммерческого канала prostor-app.**

Telegram mini-app закрыт для коммерции в РФ. Клиенты Аквафор-Pro
уже привыкли искать оборудование по фото и описанию через mini-app —
это поведенческий паттерн который сложно заменить «возвращайтесь
в обычный сайт». AI-поиск делает prostor-app **функциональным
эквивалентом TG mini-app** (фото + текст без формальных категорий) +
добавляет преимущество: клиент пишет обычными словами вместо
профессиональной терминологии («ржавая вода» вместо «коллоидное
железо», «компактный белый» вместо «настольный фильтр Тип 2»).

**Это не оптимизация существующего процесса, а сохранение канала
продаж.** Невозможно посчитать классической ROI-формулой — работает
не как «экономия часов», а как «не потеряли клиентов».

**Точка безубыточности (грубая оценка):** одна продажа фильтра Аквафор
со средним чеком от 5 000 ₽ перекрывает месяц операционных расходов
в стартовой фазе (~720 ₽/мес) или 3-4 продажи в месяц для зрелой
платформы (~18 000 ₽/мес). При том что клиентская база уже привыкла
делать заказы через mini-app — barrier to entry минимальный.

**Вторичный канал: ускорение работы менеджеров CRM.**

Менеджеры используют тот же endpoint для:
- Подбора товара клиенту в WhatsApp по фото из переписки
- Быстрого нахождения товара в каталоге при оформлении заказа
- Cross-sell услуг (монтаж, ТО) и расходников (картриджи) — они
  возвращаются в search вместе с товарами

Точная экономия зависит от volume оформляемых заказов и сложно
прогнозируется без A/B теста — оценка на 5 менеджерах по 20 подборов
в день при ускорении на 3 минуты каждый: ~88 000 ₽/мес рабочего
времени, **но это сильно зависит от ассумпций**. Honest range —
от 0 (если менеджеры не сильно используют поиск) до ~88 000 ₽/мес
(полное замещение листания каталога вручную).

**Дополнительные косвенные эффекты** (реальные значения после первой
недели в prod):
- Рост конверсии: клиент не ждёт ответа менеджера — заказывает сразу
- Снижение churn в категории «не нашёл что искал»
- Self-service: 30% клиентских запросов закрываются без саппорта =
  +10-20% capacity команды (опять же, зависит от паттернов клиентов)

Без реальных метрик MAU prostor-app точная финансовая модель — гадание.
Но **точка безубыточности тривиальна**: один заказ в месяц со средним
чеком 5 000 ₽ окупает ~150 ₽ в день AI-расходов в стартовой фазе.

## Риски и митигация

| Риск | Вероятность | Митигация |
|---|---|---|
| **Открытый каталог + анонимный traffic = abuse Vision API** ⚠️ blocker для prod | **Критическая** | Per-IP/IPv6-/64-subnet rate limit (anonymous: 1 image/min, 5 text/min) + global daily cap уже стоит ($5 Vision / $1 Embedding). Без этого один бот за час съедает весь дневной бюджет. См. tech-debt #21. |
| **Дубликат фото от одного клиента** = повторные Vision-вызовы | Средняя | SHA256-кэш image → vision_response (Redis, TTL 24ч). Ожидаемая экономия 30-50% на image. До запуска реализовать. |
| **152-ФЗ (хранение PII в РФ)** при выходе на прод | Высокая | Split-архитектура: каталог + LLM-gateway в EU, пользовательские данные с PII в РФ. План в ADR-007. |
| **Качество распознавания редких брендов** | Низкая | Промпт уже валидирован на 6 типах фото включая edge-cases. Итеративная доработка по фактическим данным. |
| **Неизвестный реальный % image vs text запросов** | Средняя | Вилка прогноза 33% — скорректируем после первой недели после запуска. Worst case (100% image) при 5000 поисков/день = ~36 000 ₽/мес — всё ещё в порядке для платформы. |
| **Зависимость от Anthropic / OpenAI** | Средняя | Архитектура multi-provider — переключение на Cohere / Ollama делается в одной ноде Flowise без изменения кода. |
| **Блокировка API из РФ** | Высокая | Подтверждена работа через корпоративный VPN. Production выйдет в EU-зоне (вне зоны блокировок). |

### Что нужно сделать ДО публичного запуска на клиентов

1. **Per-IP/IPv6-subnet rate limit** на endpoint'ы поиска (`/catalog/search/text`, `/catalog/search/image`, `/catalog/search`). Anonymous лимиты: text 30/min, image 3/min. Authenticated клиент — × 3.
2. **SHA256-кэш image-запросов** — Redis ключ `slovo:vision:cache:<sha256>` с TTL 24ч → результат vision_response.
3. **UX-loader при image-search** (6-7 сек на Vision processing) — спиннер с прогрессом или skeleton-карточек, иначе клиент жмёт «обновить» и удваивает cost.
4. **Алерт на budget-cap exhaustion** — уже срабатывает 503 на endpoint'е, но добавить Telegram/email-нотификацию админу что лимит достигнут (значит либо abuse, либо органический рост — нужно решение по увеличению лимита).

## Резюме

**Затраты для компании:**
- Разработка — **0 ₽** (ведётся как эксперимент / pet-project в свободное время разработчика)
- Реально потрачено за 9 дней работы с AI — **$0,49 ≈ 39 ₽** (фактический billing OpenAI + Anthropic, включая Phase 2 augmentation 155 товаров)
- Прогноз стартовой базы (миграция с TG mini-app) — **~720 ₽/мес**
- Прогноз активного пилота (полная клиентская база prostor-app) — **~3 600 ₽/мес**
- Прогноз зрелой платформы — **~18 000 ₽/мес**

**Главная ценность — сохранение канала продаж:**
- Telegram mini-app закрыт для коммерции в РФ
- Клиенты Аквафор-Pro привыкли искать оборудование по фото и описанию через mini-app
- AI-поиск делает prostor-app **функциональным эквивалентом TG mini-app** + добавляет преимущество (обычные слова вместо профессиональной терминологии)
- Без миграции на собственную платформу — теряем существующий поток клиентов

**Точка безубыточности:**
- Стартовая фаза (~720 ₽/мес): **одна продажа фильтра в месяц** (средний чек от 5 000 ₽) полностью окупает AI-расходы
- Активная фаза (~3 600 ₽/мес): 1-2 продажи в месяц
- Зрелая платформа (~18 000 ₽/мес): 3-4 продажи в месяц

**Дополнительные эффекты** (точные значения после первой недели в prod):
- Менеджеры CRM экономят время на подборе товара по фото в WhatsApp от клиентов
- Клиенты не звонят в саппорт по «найти аналог» — разгружают команду
- Снижение барьера для клиентов которые не знают терминологии каталога

**Срок до публичного запуска:** 1-2 недели после approve направления.
Backend pre-launch hardening полностью закрыт (per-IP throttle, image-cache,
budget-cap, Telegram alert). Остаётся UX-loader для image-search со
стороны Петра и опционально webhook-trigger для мгновенной синхронизации
с CRM (1-2 вечера разработки).

**Нужны решения по 3 пунктам перед публичным запуском:**

1. **Бюджет на AI-расходы:** текущий cap $5/день Vision (~150 ₽) рассчитан на dev-нагрузку. Для активного пилота 1000 запросов/день нужно один из вариантов:
   - **Вариант A:** расширить cap до $20-50/день (600-1 500 ₽) — комфортный запас, защита от abuse через budget-cap-alert (Telegram).
   - **Вариант B:** стартовать со строгими anonymous-лимитами (text 30/min, image 3/min) и постепенно ослаблять по мере накопления реальных данных.
   - Оба варианта закрывают анти-abuse, разница в скорости ramp-up.

2. **Сроки запуска:** Phase 1 + Phase 2 закрыты. Остаётся 1-2 вечера на webhook-trigger (мгновенная реиндексация после изменений в CRM вместо ожидания 4ч cron'а). Запускаем сразу с cron'ом или дожидаемся webhook?

3. **152-ФЗ архитектура:** для production deploy нужно решение по split-архитектуре РФ (PII клиентов) + EU (LLM-gateway). Готовый план в ADR-007. Утверждаем сейчас или Phase 3 (после первой недели в prod)?

**Operational costs вписываются в любой бюджет** — речь о суммах меньше стоимости одного обеда в неделю даже при зрелой платформе.
