# Умный каталог Аквафор-Pro: поиск по фото и тексту

> Executive summary для руководителя. Технические детали — `docs/features/vision-catalog-search.md`.

---

## Что строим

AI-сервис поиска оборудования и услуг водоочистки по **фотографии** или **текстовому запросу** на естественном языке. Встраивается в `prostor-app` (наш веб + Telegram Mini App) и `crm-aqua-kinetics-back` (CRM для менеджеров и инженеров).

**Сценарий №1.** Менеджер получает от клиента фото фильтра в WhatsApp — загружает в CRM, через 3 секунды видит подходящий товар из каталога Аквафор-Pro с ценой, нужными расходниками и услугой монтажа.

**Сценарий №2.** Клиент пишет в чате *«фильтр для жёсткой воды у дачи»* — система находит подходящие умягчители и обратные осмосы, не требуя точного знания названий моделей.

**Сценарий №3** (Phase 2). Клиент описывает проблему *«у меня жёлтая вода»* — AI диагностирует *«железо»*, подбирает обезжелезиватель и предлагает заказать анализ воды.

## Бизнес-ценность

| Что улучшается | Как |
|---|---|
| **Скорость подбора товара** | С 3-5 минут листания каталога до 2-3 секунд AI-поиска |
| **Конверсия лидов** | Быстрый ответ клиенту увеличивает вероятность сделки на 15-30% (исследования B2B) |
| **Cross-sell** | Каждый подобранный фильтр сразу предлагает связанные услуги (монтаж, ТО) и расходники (картриджи) — рост среднего чека |
| **Самообслуживание клиента** | Клиент может сам подобрать оборудование через сайт без участия менеджера — снижает нагрузку на отдел продаж |
| **Обучение новых сотрудников** | Семантический поиск работает даже без знания точных артикулов — стажёры продают с первого дня |
| **Накопление knowledge base** | Каждое распознавание — это AI-описание товара. Со временем накапливается богатый текстовый dataset для будущих фич (диагностика, рекомендации) |

## Технологический стек

| Компонент | Назначение | Лицензия |
|---|---|---|
| **Claude API (Vision)** | Распознавание фото оборудования | Pay-as-you-go ~$0.003/фото |
| **OpenAI Embeddings** | Семантический поиск (превращает текст в математический вектор для сравнения) | Pay-as-you-go ~$0.02/1М токенов |
| **Flowise** | Низкокодовый orchestrator для AI-pipeline (визуальное управление промптами и моделями) | Open-source, self-hosted |
| **PostgreSQL + pgvector** | Хранение каталога и векторных представлений | Open-source |
| **NestJS + Prisma + Docker** | Backend и инфраструктура (наш существующий стек) | Open-source |

Разработка ведётся в свободное время, расходов на нее нет — соответственно в финансовом плане учитываются только реальные **операционные** затраты на AI-сервисы.

## Финансовая модель

### Что компания платит

**Только operational costs внешних AI-провайдеров.** Инфраструктура (сервер, БД, Docker) уже включена в существующий стек CRM — incremental затрат нет. Разработка ведётся в режиме эксперимента / pet-project, без оплаты часов разработчика.

### Индексация каталога — разово и инкрементально

Каталог Аквафор-Pro (~500 товаров с фото) индексируется **один раз** при старте. Далее пересчитываются только изменившиеся товары.

| Операция | Стоимость |
|---|---|
| Первичная индексация всего каталога | **~340 ₽** разово |
| Инкрементальное обновление (5-10 товаров в день меняются) | **20-100 ₽** в месяц |

### Поисковые запросы от пользователей — runtime cost

| Тип запроса | Стоимость за запрос | Текущий объём (пилот) | Cost/мес |
|---|---|---|---|
| Поиск текстом | ~$0.0000004 (≈ 0,00004 ₽) | 100 запросов/день | ~0,1 ₽ |
| Поиск по фото | ~$0.003 (≈ 0,3 ₽) | 50 запросов/день | ~340 ₽ |

### Ежемесячный итог

| Категория | ₽/мес |
|---|---|
| Операционные AI-вызовы (Vision + Embeddings) | **400-800** |
| Инфраструктура (уже оплачена в составе CRM) | 0 (incremental) |
| **Итого** | **400-800 ₽/мес** |

### Прогноз роста стоимости при масштабировании

| Объём поисков в день | Cost/мес |
|---|---|
| 100 (текущий пилот) | **~400 ₽** |
| 1 000 (активные продажи на сети дилеров) | ~3 000 ₽ |
| 10 000 (серьёзный SaaS) | ~30 000 ₽ |

При 10-кратном росте объёма стоимость остаётся **в районе 30 тыс ₽/мес** — это копейки относительно сопутствующего роста выручки. Технологическая дорожная карта оптимизации (кеширование повторных запросов, локальные embedding-модели на нашем сервере) может снизить эту цифру в 5-10 раз.

### ROI — окупаемость

Сценарий: один менеджер делает 20 подборов товаров в день. Если каждый подбор ускоряется на 3 минуты — это **1 час экономии в день**, или ~22 часа в месяц на менеджера.

При 5 менеджерах: 110 часов/мес × 800 ₽/час (полная стоимость рабочего времени) = **~88 000 ₽/мес экономии рабочего времени** против **400-800 ₽/мес** реальных затрат на AI.

**ROI: ~100x уже на 5 менеджерах** в первый месяц после запуска.

Дополнительная нелинейная выгода (рост конверсии, увеличение среднего чека через cross-sell услуг и расходников) — измеряем после запуска через A/B-тест.

## Сроки

| Фаза | Содержание | Срок |
|---|---|---|
| **Phase 0** ✅ выполнено | Эксперименты с AI-распознаванием фото (Flowise + Claude Vision), валидация на 6 тестовых фотографиях | 1 день |
| **Phase 1** в работе | Реализация интеграции slovo + crm-aqua-kinetics-back, индексация каталога, поисковые endpoints (PR5-PR8) | 2-3 недели в текущем темпе разработки (вечерами после основной работы) или 1 неделя full-time |
| **Phase 2** | Запуск в prostor-app, A/B тестирование, тюнинг промптов | 1 неделя |

Общий MVP до запуска первым менеджерам — **3-5 недель** в текущем темпе.

## Риски и митигация

| Риск | Вероятность | Митигация |
|---|---|---|
| **152-ФЗ (хранение PII в РФ)** при выходе на прод | Высокая | Split-архитектура: каталог + LLM-gateway в EU, пользовательские данные с PII в РФ. План в ADR-007. |
| **Качество распознавания редких брендов** | Низкая | Промпт уже валидирован на 6 типах фото включая edge-cases. Итеративная доработка по фактическим данным. |
| **Рост стоимости при масштабе** | Низкая | Заложенные оптимизации (Redis-кэш, локальные модели Ollama) могут снизить cost в 5-10 раз. |
| **Зависимость от Anthropic / OpenAI** | Средняя | Архитектура multi-provider — переключение на Cohere / Ollama делается в одной ноде Flowise без изменения кода. |
| **Блокировка API из РФ** | Высокая | Подтверждена работа через корпоративный VPN. Production выйдет в EU-зоне (вне зоны блокировок). |

## KPI для измерения успеха

После запуска в prostor-app измеряем:

| Метрика | Целевое значение |
|---|---|
| Среднее время подбора товара (от запроса клиента до КП менеджера) | Снижение в 5-10 раз |
| Конверсия лидов в КП | Рост на 15-30% |
| Доля запросов закрытых через AI-поиск (без участия менеджера) | 30-50% к концу Phase 2 |
| Средний чек заказа с AI cross-sell услуг/расходников | Рост на 10-20% |
| Время онбординга нового менеджера до полной самостоятельности | Снижение на 30-50% |
| Удовлетворённость клиентов (NPS на этапе ответа) | Рост на 10+ пунктов |

## Резюме

**Затраты для компании:**
- Разработка — **0 ₽** (ведётся как эксперимент / pet-project в свободное время разработчика)
- Ежемесячно — **~400-800 ₽** operational costs на Claude Vision + OpenAI Embeddings
- Разовые — **~340 ₽** на первичную индексацию каталога

**Выгоды (консервативная оценка на 5 менеджеров):**
- **~88 000 ₽/мес** экономии рабочего времени
- Рост конверсии и среднего чека (нелинейная выгода через cross-sell)
- Накопление AI-описаний каталога — фундамент для следующих фич (диагностика проблем воды, рекомендательные системы)

**ROI: ~100x уже на пилоте с 5 менеджерами**, далее рост с увеличением сети дилеров.

**Срок до пилотного запуска первым менеджерам:** 3-5 недель в текущем темпе разработки (вечерами после основной работы).

**Решение по запуску:** согласование пилота с 2-3 дилерскими менеджерами для замеров KPI на реальных данных. Operational costs вписываются в любой бюджет — речь о суммах меньше стоимости одного обеда в неделю.
