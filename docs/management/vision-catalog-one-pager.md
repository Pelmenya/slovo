## Что работает сейчас

**Поддерживаются 7 паттернов поиска**, проверены на полном каталоге (155 товаров):

1. **По артикулу** — менеджер вводит «DWM-101S» → товар + связанные комплекты
2. **Описание проблемы** — клиент пишет «фильтр от ржавой воды» → находится обезжелезиватель
3. **Услуги к товарам** — «установка фильтра под мойку» → товар + монтажные услуги
4. **Визуальные слова** — «компактный белый с прозрачными колбами» → подходящие модели
5. **Фото товара** — клиент фотографирует свой фильтр → находится оригинал и аналоги
6. **Защита от случайных фото** — котики/документы возвращают «AI не нашёл оборудование»
7. **Фото + уточнение** — «самый компактный» + фото → ближайший компактный аналог

**Время ответа:** 95% запросов <1.5 секунды. Текстовый поиск стоит **доли копейки**, поиск по фото — **50 копеек** с кэшем повторов на 24 часа.

## Что нужно решить руководителю

Перед публичным запуском на клиентов prostor-app — 3 пункта:

### 1. Бюджет AI-расходов

Текущий лимит **$5/день** (~150 ₽) — для dev-нагрузки. На активный пилот (1000 запросов/день) нужно один из вариантов:

- **Вариант A:** расширить лимит до **$20-50/день** (600-1 500 ₽) с комфортным запасом
- **Вариант B:** стартовать со строгими лимитами для анонимов (3 image/мин, 30 text/мин) и постепенно ослаблять

Оба защищают от злоупотреблений, разница только в скорости роста.

### 2. Сроки запуска

Backend готов сейчас. Опциональный enhancement — **webhook-trigger** (мгновенная переиндексация при изменениях в CRM вместо ожидания 4 часа). Можно:

- **Запускать сейчас** с переиндексацией каждые 4 часа — простое решение
- **Дождаться webhook** (1-2 вечера разработки) — мгновенная синхронизация

Frontend (UX-loader для image-search) — задача **Петра**, ~1-2 дня после approve направления.

### 3. Архитектура под 152-ФЗ

Для production нужно решение по разделению:
- **РФ-сегмент**: данные клиентов (PII)
- **EU-gateway**: LLM-вызовы (Anthropic / OpenAI заблокированы из РФ)

Готовый план в ADR-007. Можно:
- **Утвердить сейчас** — параллельно с Phase 2 запуском
- **Отложить в Phase 3** — после первой недели в prod, когда соберём реальные метрики

## Главный аргумент

**prostor-app — замена Telegram mini-app**, который закрыт для коммерции в РФ. Без миграции теряем канал продаж клиентов которые уже привыкли искать оборудование по фото.

AI-поиск делает миграцию **функциональным эквивалентом** Telegram mini-app — клиент сразу пишет на естественном языке или фотографирует, без поиска по дропдаунам и категориям.
