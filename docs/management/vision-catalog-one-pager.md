# Vision-catalog · One-pager

> AI-поиск товаров в каталоге Аквафор-Pro по тексту и фото для **клиентов
> на prostor-app** и менеджеров CRM. Готов к подключению клиентского фронта.
>
> **UX готов** — 3 состояния mobile flow визуализированы в
> [vision-catalog-ux-mockup.html](vision-catalog-ux-mockup.html)
> ([live-просмотр](https://raw.githack.com/Pelmenya/slovo/main/docs/management/vision-catalog-ux-mockup.html))
> — не нужно объяснять «как это будет в руках клиента», просто открыть.

---

## Что это даёт

**Главное — сохранение коммерческого канала prostor-app.** Telegram
mini-app закрыт для коммерции в РФ, но клиенты уже привыкли искать
товары по фото и описанию. Без миграции на собственную платформу теряем
этих клиентов. AI-поиск делает prostor-app **функциональным эквивалентом**
TG mini-app — клиент пишет на естественном языке или фотографирует,
без выбора по дропдаунам и категориям.

**Дополнительно** — менеджеры CRM используют тот же endpoint для
быстрого подбора товаров клиентам, оформления заказов в WhatsApp.

## Что работает сейчас

**Поддерживаются 7 паттернов поиска**, проверены на полном каталоге
(155 товаров Аквафор-Pro):

1. **По артикулу** — клиент или менеджер вводит «DWM-101S» → товар + комплекты
2. **Описание проблемы** — клиент пишет «фильтр от ржавой воды» → находится обезжелезиватель
3. **Услуги к товарам** — «установка фильтра под мойку» → товар + монтажные услуги
4. **Визуальные слова** — «компактный белый с прозрачными колбами» → подходящие модели
5. **Фото товара** — клиент фотографирует свой фильтр → находится оригинал и аналоги
6. **Защита от случайных фото** — котики/документы возвращают «AI не нашёл оборудование»
7. **Фото + уточнение** — «самый компактный» + фото → ближайший компактный аналог

**Время ответа:** 95% запросов <1.5 секунды. Текстовый поиск — **доли копейки**,
поиск по фото — **~50 копеек** с кэшем повторов на 24 часа.

## Прогноз расходов и точка безубыточности

Без реальной статистики MAU prostor-app точные числа — гадание.
Честный диапазон по сценариям:

| Сценарий | Запросов/день | Расходы/мес |
|---|---|---|
| **Стартовая база** (миграция первых клиентов с TG) | 200 | ~720 ₽ |
| **Активный пилот** (полная клиентская база prostor-app) | 1 000 | ~3 600 ₽ |
| **Зрелая платформа** | 5 000 | ~18 000 ₽ |

**Точка безубыточности:** одна продажа фильтра Аквафор со средним чеком
от 5 000 ₽ окупает месяц операционных расходов в стартовом сценарии.
В зрелом — 3-4 продажи в месяц.

Для сравнения: реальный budget-cap сейчас выставлен на $5/день (~150 ₽,
~4 500 ₽/мес) — это по сути верхняя граница для стартовой и активной фаз.

## Что нужно решить руководителю

### 1. Бюджет на AI-расходы

Текущий лимит **$5/день** (~150 ₽). Для активного пилота нужно
один из вариантов:

- **Вариант A:** расширить до **$20-50/день** (600-1 500 ₽) с запасом
- **Вариант B:** стартовать со строгими лимитами для анонимов
  (3 image/мин, 30 text/мин) и ослаблять по мере роста traffic'а

Оба защищают от злоупотреблений, разница только в скорости ramp-up.

### 2. Сроки запуска

Backend готов сейчас. Опциональный enhancement — **webhook-trigger**
(мгновенная переиндексация при изменениях в CRM вместо ожидания 4 часа).

- **Запускать сейчас** с переиндексацией каждые 4 часа — простое решение
- **Дождаться webhook** (1-2 вечера разработки) — мгновенная синхронизация

Frontend (UX-loader для image-search) — задача **Петра**, ~1-2 дня
после approve направления.

### 3. Архитектура под 152-ФЗ

Для production deploy нужно решение по разделению:
- **РФ-сегмент**: данные клиентов (PII)
- **EU-gateway**: LLM-вызовы (Anthropic / OpenAI заблокированы из РФ)

Готовый план в ADR-007. Можно:
- **Утвердить сейчас** — параллельно с Phase 2 запуском
- **Отложить в Phase 3** — после первой недели в prod, когда соберём
  реальные метрики

## Главный аргумент

**prostor-app — замена Telegram mini-app, который закрыт для коммерции в РФ.**
Без миграции теряем канал продаж клиентов которые уже привыкли искать
оборудование по фото.

AI-поиск делает миграцию **функциональным эквивалентом** Telegram mini-app
с дополнительным преимуществом: клиент пишет на естественном языке или
фотографирует, без поиска по дропдаунам и категориям. Это **снижает
барьер входа для клиентов** которые не знают профессиональной терминологии
(«ржавая вода» вместо «коллоидное железо»).
