---
description: Запускает 5 reviewer-агентов параллельно на текущие изменения (git diff main...HEAD) и возвращает сводный отчёт.
---

Запусти параллельно (одним сообщением, несколько `Agent` tool-use блоков) следующих агентов на текущие изменения branch'а:

1. **architect-reviewer** — соответствие ADR, границы модулей, RAG-архитектура
2. **nestjs-code-reviewer** — стиль TS/NestJS, DTO, валидация, Swagger
3. **prisma-pgvector-reviewer** — схема, миграции, raw queries, индексы
4. **llm-integration-reviewer** — Anthropic SDK, prompts, caching, observability
5. **security-auditor** — секреты, JWT, rate limit, SQL injection, PII

## Как задать скоуп каждому агенту

Каждому агенту в prompt передай:

- Branch / diff: если пользователь не указал явный скоуп в `$ARGUMENTS`, используй `git diff main...HEAD` и `git status`.
- Если `$ARGUMENTS` содержит номер PR — `gh pr diff <N>`.
- Если `$ARGUMENTS` содержит путь к файлу/папке — ограничь агента этим путём.

## Что возвращаешь пользователю

После того как все 5 агентов закончат, сделай **сводный отчёт**:

```markdown
# Сводный ревью

**Скоуп:** <что ревьювили>

## 🔴 Критичное (всего N)
- [architect] `file:line` — …
- [nestjs] `file:line` — …
- [security] `file:line` — …

## 🟡 Важное (всего N)
- [prisma] `file:line` — …

## 🟢 Рекомендации (всего N)
- …

## ✅ Что сделано хорошо
- …

## Следующие шаги
Предложи порядок исправлений (сначала критичное, потом важное) и спроси у пользователя, с чего начать.
```

Сгруппируй замечания по приоритету (не по агенту). В квадратных скобках — какой агент нашёл (для контекста).

Если агент вернул «замечаний нет» — упомяни его в одну строку: `✅ architect-reviewer — замечаний нет`.

## Если $ARGUMENTS пустой

Прогоняй на `git diff main...HEAD`. Если diff пустой (branch == main, рабочая копия чистая) — ответь пользователю «нет изменений для ревью» и спроси, какой скоуп использовать.
